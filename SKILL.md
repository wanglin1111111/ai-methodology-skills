# Build in Public & Life Proof Skill

## 技能概述

本技能专注于通过社交媒体真实分享个人故事与构建过程的方法论，帮助用户与项目背后的真实人物建立情感连接，使受众不仅关注产品本身，更能共情于构建者的旅程与挑战。

**核心领域**：
- 核心理念：真实分享建立连接
- 三种参与路径设计
- 平台要求与时间安排
- 灵活性与安全准则
- 信息获取渠道

---

## 1. 核心理念：真实分享建立连接

### 1.1 传统方式与真实分享对比

```yaml
传统方式:
  方法: 单纯展示项目
  特点: 仅展示成果
  问题:
    - 连接浅层
    - 理解不深
    - 共鸣不足
    - 参与度低

真实分享:
  方法: authentically（真实地）分享
  
  分享内容:
    内容1: 个人故事
      类型: 个人经历和背景
      意义: 人物立体化
    
    内容2: 意图
      类型: 构建动机和目标
      意义: 目标透明化
    
    内容3: 构建过程起伏
      类型: 包括极度高涨和低谷时刻
      意义: 过程真实化
  
  效果:
    效果1: 更好理解
      内容: 让用户更好地理解为何成为用户
      意义: 建立深度理解
    
    效果2: 参与旅程
      内容: 参与到构建产品的旅程中
      意义: 形成共同经历
    
    效果3: 打破不理解
      内容: 打破旁观者对"构建者挣扎"的不理解
      意义: 消除隔阂
    
    效果4: 建立共鸣
      内容: 建立更深层次的共鸣
      意义: 情感连接

核心价值:
  价值1: 透明度
    内容: 透明度建立信任
    意义: 增强可信度
  
  价值2: 真实性
    内容: 真实性建立连接
    意义: 情感共鸣
  
  价值3: 共同旅程
    内容: 共同旅程建立参与感
    意义: 深度参与
```

### 1.2 真实分享的要素

```yaml
要素1: 个人故事
  内容: 分享个人背景和经历
  目的: 让用户了解构建者
  方法: 真实叙述
  
  特点:
    - 真实不虚构
    - 个人化表达
    - 情感真挚
    - 易于共情

要素2: 构建意图
  内容: 分享构建的动机和目标
  目的: 让用户理解为何而建
  方法: 意图阐述
  
  特点:
    - 动机明确
    - 目标透明
    - 价值清晰
    - 理由充分

要素3: 过程起伏
  内容: 分享构建过程的起伏
  目的: 展示真实过程
  方法: 过程记录
  
  特点:
    - 包含高涨时刻
    - 包含低谷时刻
    - 真实不掩饰
    - 过程完整

要素4: 挣扎与挑战
  内容: 分享遇到的挣扎和挑战
  目的: 打破完美假象
  方法: 困难分享
  
  特点:
    - 承认困难
    - 展示挣扎
    - 分享解决
    - 真实呈现

价值总结:
  - 打破旁观者误解
  - 建立深度理解
  - 形成情感连接
  - 创造共同旅程
```

## 2. "公开构建与生活证明"三种参与路径

### 2.1 路径一：深度跟随实验进程

```yaml
适合人群:
  人群: 对特定实验高度感兴趣的人
  示例: CGN实验或睡眠研究
  
参与方式:
  方式: 跟随实验负责人
  示例: Rebecca的生物黑客实践或Frank的挑战经历
  
  参与内容:
    内容1: 实验设计
      阶段: 设计阶段
      记录: 设计思路和约束
    
    内容2: 执行过程
      阶段: 执行阶段
      记录: 实际执行情况
    
    内容3: 数据收集
      阶段: 收集阶段
      记录: 数据获取过程

记录重点:
  重点1: 科学过程背后的约束
    内容: 展示实验约束条件
    意义: 真实性体现
  
  重点2: 遇到的问题
    内容: 记录遇到的问题
    意义: 困难透明化
  
  重点3: 未达预期的部分
    内容: 记录未达预期的情况
    意义: 失败不掩饰

价值体现:
  价值: 呈现比实验室报告更宏观、更真实的故事全貌
  
  对比:
    实验室报告: 
      特点: 技术性强、结果导向
      问题: 缺少过程故事
    
    真实分享:
      特点: 宏观视角、真实过程
      优势: 故事全貌完整

路径特点:
  - 科学性强
  - 过程完整
  - 真实透明
  - 故事丰富
```

### 2.2 路径二：基于个人疑问的日常探索

```yaml
适合人群:
  人群: 有特定好奇心的人
  示例: 探究中医与现代医学/西方资源的联系

参与方式:
  方式: 创意性的"实验设计"
  
  参与内容:
    内容1: 提出问题
      行动: 提出自己的问题或提案
      意义: 问题驱动
    
    内容2: 每日记录
      行动: 每日发帖记录发现的新知
      意义: 持续探索
    
    内容3: 知识梳理
      行动: 梳理知识并回答自己的疑问
      意义: 知识整合

方式特点:
  特点1: 兼具硬科学的严谨性
    内容: 科学方法严谨
    意义: 可靠性高
  
  特点2: 内容创作的灵活性
    内容: 创作形式灵活
    意义: 表达自由

参与价值:
  - 满足好奇心
  - 系统探索
  - 知识积累
  - 内容创作

路径特点:
  - 问题驱动
  - 持续探索
  - 知识梳理
  - 兼顾严谨与灵活
```

### 2.3 路径三：记录生活方式与成员体验

```yaml
适合人群:
  人群: 处于探索阶段或参与度较松散的人群

参与方式:
  身份: 以长寿营成员或WSHanghai成员的身份
  行动: 日常记录自己的生活体验和对长寿领域的感悟

参与内容:
  内容1: 生活体验记录
    类型: 日常生活记录
    意义: 生活分享
  
  内容2: 长寿领域感悟
    类型: 领域思考和感受
    意义: 观点分享

参与特点:
  特点1: 门槛低
    内容: 参与门槛低
    意义: 易于参与
  
  特点2: 灵活性高
    内容: 参与灵活
    意义: 自由度高
  
  特点3: 探索性
    内容: 探索阶段
    意义: 学习成长

路径价值:
  - 轻松参与
  - 生活记录
  - 感悟分享
  - 社区参与

适合场景:
  - 初次参与者
  - 时间有限者
  - 探索兴趣者
  - 社区成员
```

## 3. 平台要求与时间安排

### 3.1 平台要求

```yaml
主要平台:
  平台1: X（原Twitter）
    要求: 必须每日发帖
    平台特点: 国际化、实时性强
    
  平台2: 小红书（Red Notebook）
    要求: 必须每日发帖
    平台特点: 本地化、图文并茂

发帖要求:
  频率: 每日发帖
  平台: 两个平台都需发布
  强制性: 必须完成

平台价值:
  - 双平台覆盖
  - 受众多元化
  - 内容形式多样
  - 影响力扩大
```

### 3.2 关键时间节点

```yaml
时间安排:
  节点1: 账号设置截止
    时间: 16日前
    内容: 完成X和小红书账号的设置
    重要性: 必须完成
  
  节点2: 研讨会
    时间: 16日
    内容: 教授如何构建社交媒体策略
    目的: 能力建设
  
  节点3: 挑战赛开始
    时间: 19日
    内容: 官方"公开构建生活证明"挑战赛正式开始计数
    重要性: 正式启动

时间规划建议:
  建议1: 提前准备
    内容: 现在就开始拍摄、记录和剪辑内容
    目的: 无需等到正式挑战开始
    意义: 提前积累素材
  
  建议2: 减轻压力

    意义: 避免断更
  
  建议3: 确保持续性
    内容: 确保持续更新
    目的: 保持参与
    意义: 完成挑战
```

注意事项:

## 4. 灵活性与安全准则

### 4.1 提案机制

```yaml
提案背景:
  背景: 参与者有不同的创作节奏
  需求: 适应不同创作方式

提案类型:
  类型: 长周期、高编辑质量的周更视频
  对比: 非日更形式
  
  特点:
    特点1: 长周期
      内容: 制作周期较长
      意义: 深度内容
    
    特点2: 高编辑质量
      内容: 编辑质量高
      意义: 精品内容
    
    特点3: 周更
      内容: 每周更新
      意义: 不同节奏

提案流程:
  步骤1: 提交个人提案
    截止: 19日
    内容: 提交创作计划
  
  步骤2: 团队审核
    对象: 团队
    内容: 审核提案内容
  
  步骤3: 调整考核
    对象: 团队
    内容: 根据提案内容调整考核方式

提案价值:
  - 灵活适应不同创作者
  - 尊重创作节奏
  - 保证内容质量
  - 个性化考核
```

### 4.2 安全红线

```yaml
安全原则:
  原则: 严禁任何可能伤害自己或他人的危险行为

禁止内容:

    内容: 可能危害神经系统的活动
    风险: 健康风险
    级别: 严禁
  
  禁止2: 侵入性程序
    内容: 侵入性的医疗或生物程序
    风险: 安全风险
    级别: 严禁

责任说明:
  说明: 相关责任由个人承担
  意义: 责任明确
  重要性: 安全保障

合规要求:
  要求: 内容必须遵守Mushkai和Longecka的相关指南
  
  指南要求:
    要求1: 追求优质内容
      内容: 在追求优质内容的同时
      意义: 质量保证
    
    要求2: 保证安全性
      内容: 保证安全性
      意义: 安全第一

安全价值:
  - 保护参与者安全
  - 规范内容创作
  - 明确责任边界
  - 确保合规运营
```

## 6. 使用本技能的场景

**触发条件**（当以下情况时使用此技能）：

1. 设计"公开构建"分享策略

3. 制定社交媒体发布计划
4. 建立真实分享内容框架
5. 平台配置与时间规划
6. 用户提到"公开构建"、"生活证明"、"真实分享"、"Build in Public"

**典型任务**：
- 设计真实分享策略
- 选择三种参与路径之一
- 制定双平台发布计划
- 建立安全合规内容标准
