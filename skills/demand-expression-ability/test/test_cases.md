# Test Cases for demand-expression-ability

## Test Suite: Demand Identification

### Test 1: Clarify Ambiguous Request
**Input**: "我想要更好的工作"

**Expected Output**:
- Probe for specifics: "better" definition
- Identify dimensions: salary, role, industry, culture
- Guide through prioritization
- Transform to clear demand statement

**Validation Points**:
- [ ] Specifics extracted
- [ ] Dimensions identified
- [ ] Clear demand statement generated

### Test 2: Hidden Demand Discovery
**Input**: "我不知道自己想要什么"

**Expected Output**:
- Provide exploration framework
- Ask reflective questions
- Identify patterns from past choices
- Surface underlying values and needs

**Validation Points**:
- [ ] Framework provided
- [ ] Questions insightful
- [ ] Patterns identified

### Test 3: Multi-Stakeholder Demand
**Input**: "我们团队需要新工具"

**Expected Output**:
- Identify all stakeholders
- Elicit each party's needs
- Find common ground
- Synthesize unified demand

**Validation Points**:
- [ ] Stakeholders identified
- [ ] Individual needs captured
- [ ] Unified demand articulated

## Test Suite: Demand Expression

### Test 4: Written Expression
**Input**: "帮我写一封需求邮件给供应商"

**Expected Output**:
- Structure: context, specific requirements, constraints, timeline
- Language: clear, specific, measurable
- Tone: professional but firm
- Follow-up: clear next steps

### Test 5: Verbal Expression
**Input**: "我要在会议上提出我的需求"

**Expected Output**:
- Opening strategy
- Key points framing
- Objection anticipation
- Closing with commitment

### Test 6: Visual Expression
**Input**: "如何用图表表达我的需求？"

**Expected Output**:
- Visual format recommendation
- Key elements to include
- Data presentation tips
- Clarity enhancement techniques
