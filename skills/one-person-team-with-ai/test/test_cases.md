# Test Cases for one-person-team-with-ai

## Test Suite: Team Structure Design

### Test 1: Role Analysis
**Input**: "我是一个自由设计师，AI可以帮我做哪些工作？"

**Expected Output**:
- Analyze current workflow
- Identify AI-replaceable tasks: initial drafts, variations, resizing
- Identify human-must tasks: creative direction, client communication, final judgment
- Propose AI team structure

**Validation Points**:
- [ ] Workflow breakdown clear
- [ ] AI vs human roles defined
- [ ] Practical team structure proposed

### Test 2: Tool Selection
**Input**: "我应该选择哪些AI工具来组建我的团队？"

**Expected Output**:
- Assess user's domain and needs
- Recommend specific tools by function:
  - Content: GPT-4, Claude
  - Design: Midjourney, Stable Diffusion
  - Code: GitHub Copilot, Cursor
  - Research: Perplexity, Elicit
- Provide integration workflow

**Validation Points**:
- [ ] Tool recommendations relevant
- [ ] Coverage of key functions
- [ ] Integration approach practical

### Test 3: Workflow Design
**Input**: "帮我设计一个AI辅助的工作流程"

**Expected Output**:
1. Current state mapping
2. AI integration points identification
3. New workflow diagram
4. Handoff protocols
5. Quality control checkpoints

**Validation Points**:
- [ ] All 5 elements present
- [ ] Workflow logical and executable
- [ ] Quality control included
