## 4. 最小权限实现评估

### 4.1 三维度评估

#### 维度1：资源范围

```yaml
评估项: 是否实现了资源范围限制

当前实现:
  - [ ] 仅传递特定文件引用
  - [ ] 仅传递特定目录引用
  - [ ] 传递整个主目录引用
  - [ ] 传递系统级权限

评估结果:
  评分: [1-5分]
  风险: [高/中/低]
  建议: [改进建议]

示例:
  当前做法: Agent可访问/home/user/下所有文件
  建议做法: Agent仅能访问/home/user/project-A/目录
```

#### 维度2：时间范围

```yaml
评估项: 是否实现了时间范围限制

当前实现:
  - [ ] 权限在任务完成后自动失效
  - [ ] 权限有明确过期时间
  - [ ] 权限持续有效，需手动撤销
  - [ ] 权限永久有效

评估结果:
  评分: [1-5分]
  风险: [高/中/低]
  建议: [改进建议]

示例:
  当前做法: Agent权限在会话期间持续有效
  建议做法: Agent权限在任务完成后立即失效，最长30分钟
```

#### 维度3：传播范围

```yaml
评估项: 是否实现了传播范围限制

当前实现:
  - [ ] 权限可自由委托给其他Agent
  - [ ] 权限委托需审批
  - [ ] 权限单调递减（只能委托子集）
  - [ ] 权限无法委托

评估结果:
  评分: [1-5分]
  风险: [高/中/低]
  建议: [改进建议]

示例:
  当前做法: AgentA可将权限委托给AgentB，AgentB可再委托给AgentC
  建议做法: AgentA只能委托权限子集给AgentB，权限单调递减
```

### 4.2 综合评分


## 6. 安全加固建议

### 6.1 立即实施（P0）

```yaml
高风险措施:
  1. 限制文件访问范围
     - 仅授权任务所需的特定文件/目录
     - 禁止访问用户主目录
     - 禁止访问系统配置文件
  
  2. 添加权限过期机制
     - 设置合理的过期时间（如30分钟）
     - 任务完成后自动撤销
     - 长时间任务定期刷新
  
  3. 实施审计日志
     - 记录所有能力创建
     - 记录所有能力使用
     - 记录所有能力撤销
     - 记录所有异常访问
```

### 6.2 短期实施（P1，1个月内）

```yaml
中风险措施:
  1. 实现能力传递机制
     - 实现权限单调递减
     - 禁止权限增强
     - 禁止权限伪造
  
  2. 添加异常检测
     - 检测异常访问模式
     - 检测权限滥用
     - 自动告警和阻断
  
  3. 安全测试
     - 模拟混淆代理攻击
     - 模拟恶意提示词攻击
     - 模拟横向移动攻击
```

### 6.3 长期优化（P2，3个月内）

```yaml
低风险措施:
  1. 完善OCaps实现
     - 支持更多资源类型
     - 优化性能
     - 添加更多语言支持
  
  2. 安全培训
     - 开发者安全培训
     - 安全编码规范
     - 安全review流程
```
