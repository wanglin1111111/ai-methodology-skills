## 方法论框架

### 协作能力培养四步法

**第一步：工具入门**
- 学习2-3个核心AI工具
- 了解基本功能和局限
- 练习简单任务

**第二步：场景应用**
- 识别工作中的AI应用场景
- 用AI完成实际工作任务
- 记录效果并优化

**第三步：流程重构**
- 设计"人+AI"协作流程
- 明确人机分工边界
- 建立质量审核机制

**第四步：能力内化**
- 将AI协作变成习惯
- 持续优化提示词
- 探索高级应用场景

## 行动检查清单

- [ ] 选择2-3个核心AI工具深入学习
- [ ] 学习提示词（Prompt）编写技巧

### 场景应用

- [ ] 列出工作中的20个任务
- [ ] 标记可以用AI辅助的任务
- [ ] 每周用AI完成至少5个任务
- [ ] 记录使用效果和问题

### 流程重构

- [ ] 设计"人定义，AI执行"的工作流
- [ ] 明确哪些环节人必须参与
- [ ] 建立AI输出的审核标准
- [ ] 优化协作流程

### 持续进阶

- [ ] 每月学习1个新的AI工具
- [ ] 加入AI工具使用社群
- [ ] 分享AI协作经验
- [ ] 跟踪AI技术发展

## 常见误区警示

| 误区 | 正确认知 |
|------|---------|
| ❌ "学会AI工具就能高枕无忧" | ✅ "工具会迭代，需要持续学习" |
| ❌ "AI输出可以直接用" | ✅ "AI输出需要人审核和优化" |
| ❌ "AI会取代我的思考" | ✅ "AI增强思考，但不替代思考" |
| ❌ "提示词越复杂越好" | ✅ "提示词要清晰、具体、可执行" |

## 参考案例

**案例**：产品经理的AI协作升级

**背景**：产品经理，每天写PRD、做调研

**实践**：
1. 用AI辅助PRD撰写：效率提升3倍
2. 用AI做竞品分析：时间从2天到2小时
3. 用AI生成用户访谈提纲：质量提升

**结果**：
- 每天节省3小时
- 产出质量提升
- 有更多时间做策略思考
