## 💡 核心理念

### 真实分享 vs 传统展示

\\\yaml
传统方式: 单纯展示项目
真实分享: authentically分享个人故事、意图、构建过程起伏

效果:
  - 让用户更好地理解为何成为用户
  - 参与到构建产品的旅程中
  - 打破旁观者对"构建者挣扎"的不理解
  - 建立更深层次的共鸣
\\\

### 四大要素

| 要素 | 内容 | 价值 |
|------|------|------|
| **个人故事** | 分享个人背景和经历 | 人物立体化 |
| **构建意图** | 分享构建动机和目标 | 目标透明化 |
| **过程起伏** | 包含高涨和低谷时刻 | 过程真实化 |
| **挣扎挑战** | 分享遇到的困难 | 打破完美假象 |

## 🛤️ 三种参与路径

### 路径一：深度跟随实验进程

\\\yaml
适合人群: 对特定实验高度感兴趣的人
示例: CGN实验、睡眠研究

参与方式: 跟随实验负责人
记录重点:
  - 科学过程背后的约束
  - 遇到的问题
  - 未达预期的部分

价值: 呈现比实验室报告更宏观、更真实的故事全貌
\\\

### 路径二：基于个人疑问的日常探索

\\\yaml
适合人群: 有特定好奇心的人
示例: 探究中医与现代医学的联系

参与方式: 创意性的"实验设计"
特点: 兼具硬科学严谨性与内容创作灵活性
\\\

### 路径三：记录生活方式与成员体验

\\\yaml
适合人群: 探索阶段或参与度较松散的人群
身份: 长寿营成员或WSHanghai成员
内容: 日常记录生活体验和长寿领域感悟
\\\
